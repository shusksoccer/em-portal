# Phase 4 学習到達検証シナリオ

目的:
- 高校生が `理解 -> 練習 -> 自力実践` を最後まで到達できるかを、導線と成果物の両面で検証する。

実施ルール:
- 1シナリオずつ実施し、詰まり箇所を都度ログ化する。
- 詰まりが発生したら、必ず「どのページで、何が分からなかったか」を記録する。
- 修正後は `pnpm run build` で整合確認する。

記録フォーマット（各シナリオ共通）:
- 実施日時:
- 実施者:
- 成否: 成功 / 失敗
- 詰まり箇所:
- 修正内容:
- 再確認結果:

## シナリオA: 概念理解のみ

到達目標:
- EMの対象を説明できる
- 根拠・限界・倫理の3点を言語化できる

推奨ルート:
1. `/`
2. `/intro`
3. `/curriculum`
4. `/curriculum/l1-what-is-em`
5. `/glossary/accountability`
6. `/figures/fig-learning-route`

確認項目:
- L1ページで「高探究チェック」が読めるか
- 用語と図解に1クリックで到達できるか
- 「次に何をするか」がページ内に明示されているか

成功条件:
- ルート上でリンク抜けがない
- 15分以内にL1の到達目標を説明できる

## シナリオB: ワーク実施まで

到達目標:
- 1つの授業でワーク提出準備まで完了する

推奨ルート:
1. `/intro`
2. `/curriculum/l2-how-to-observe`
3. `/worksheets/ws-l2`
4. `/faq`（詰まり確認）
5. `/curriculum/l3-how-to-describe`

確認項目:
- ワークページで所要時間・提出物・評価観点が見えるか
- ワークページから「次のアクション」に遷移できるか
- FAQ経由後に授業導線へ復帰しやすいか

成功条件:
- `ws-l2` から行き止まりにならない
- 20分想定でワーク実施手順が理解できる

## シナリオC: 自力ミニ実践まで

到達目標:
- 問い・方法・根拠・限界・倫理を含む発表計画を作る

推奨ルート:
1. `/curriculum/l6-project`
2. `/worksheets/ws-l6`
3. `/figures/fig-presentation-map`
4. `/glossary/validity`
5. `/library?stage=自力実践`（必要時）

確認項目:
- L6で最終成果物の要件が読み取れるか
- WS6で提出物が具体化されているか
- 妥当性・倫理・限界の観点が抜けずに記述できるか

成功条件:
- ミニ実践計画を1ページで提出可能な状態にできる
- ルート上で「補助資料」と「必修ルート」が混線しない
