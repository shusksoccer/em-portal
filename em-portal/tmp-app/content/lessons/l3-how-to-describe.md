## 導入：なぜ記述を整えるか

:::example 観察メモのままでは分析できない
「なんとなく重い空気」「途中で話が止まった」とだけ書かれたメモは、読み手によって意味が変わります。時系列と発話単位をそろえることで、他者と同じデータを検討できます。
:::

## 概念の説明

:::concept 3列表（時刻・発話・行為）
記述段階では、観察ログを3列に変換してデータ化します。時刻で順序を固定し、発話と行為を分離して、注釈は必要最小限にします。これにより、次の会話分析で行番号を根拠として使えます。
:::

:::transcript 3列表への変換例
1: 12:03 A「これ先生に見せる？」 / Aが紙を持ち上げる
2: 12:03 B「まだ早いかも」 / Bが首を振る
3: 12:04 A「じゃあ修正しよう」 / Aが赤ペンを出す
---
同じ秒でも順序は保持する。発話と行為を併記することで、判断の根拠が増える。
:::

## 会話分析のための記述：何を保存するか

会話分析（CA）では、発話の内容だけでなく、**タイミング・間・重なり**が重要な意味を持ちます。記述段階でこれらを消してしまうと、後の分析で使えなくなります。

:::concept Jefferson式転写記号（授業で使う最小セット）
会話分析の標準的な記録方式（Jefferson式）から、高校探究で必要な最小限の記号を使います：

| 記号 | 意味 | 例 |
|---|---|---|
| `[ ]` | 発話が重なった部分 | A「行こ[う]」 B「[あ]待って」 |
| `(0.5)` | 沈黙の秒数 | A「本当に？」(2.0) B「うん」 |
| `(.)` | ごく短い間（0.2秒以下） | 「それは(.)難しい」 |
| `?` | 上昇イントネーション | 「今日来るの?」 |
| `:` | 音の引き延ばし | 「えー:と」 |

これらを使う理由：「2秒の沈黙」は単なる間ではなく、応答の拒否・困惑・考慮中といった社会的な意味を持つ場合があるからです。
:::

:::transcript 記号を使った転写例
1: A「明日の発表、準備できてる?」
2: (1.8)
3: B「ま:あ、なんとか」
4: A「なんとか[って]」
5: B:        [あ]でも全部は終わってない」
---
2行目の1.8秒の沈黙は「すぐにYesと言えない」状況を示す。3行目「まあ」と語の引き延ばしは非優先的応答の形式。4行目Aが割り込もうとしたところ（4〜5行目）で発話の重なりが起きている。
:::

## 記述の目的：誰もが同じ場面を追えること

:::concept 再現可能性が記述の基準
よい記述とは「第三者が読んで場面を追えるか」で判断します。観察者が「分かっていること」を前提にした記述は、共有できません。以下の順序で書く習慣をつけてください：

1. **場面設定**：どこで・誰が参加しているか
2. **時系列**：何が起きたか（行番号で整理）
3. **発話と行為**：言語と非言語を分けて記録
4. **注記**：読み手が誤解しそうな点のみ補足
:::

## よくある誤り

:::caution
注釈を書きすぎて本文より長くなると、データが見えなくなります。注釈は「読み手が誤解しそうな点」に限定し、判断の結論は別欄に分けてください。

また、記号を使いすぎてデータが読めなくなることも避けてください。最初は時刻・発話・行為の3列から始め、慣れてから沈黙や重なりを加えてください。
:::

## やってみよう

:::exercise
1. 30秒分を3列で最低5行作る。
2. 重要行を1つ選び、理由を1文で書く。
3. 限界（不足情報）を1点記録する。
:::
