## 導入：「まとめ」は終わりではなく、自分の探究の入口

L1〜L5で身につけた観察・記述・分析の手法は、ここから先の**自分の問い**に使うためのものです。L6のゴールは「授業の課題を終わらせる」ことではなく、「自分が気になる日常の場面にEMを当てて、言語化できるようにする」こと。

:::example テーマが広すぎる最初の一歩
「SNSのコミュニケーションを調べたい」——これは問いではなく、関心の方向です。L6では、その関心を「どこで・誰の・どのやりとりを・どの方法で」見るかに絞り込みます。絞れば絞るほど、言えることが増えます。
:::

## ステップ1　自分のテーマを見つける

### 「気になる場面」の発見法

EMのテーマは遠くにあるのではなく、毎日の中にあります。以下の3つの問いから始めてください。

:::concept テーマ発見の3つの入口
**入口①　壊れたとき**
「なぜ？」と感じた瞬間を思い出す。誰かが「普通じゃない」ことをしたとき、何が起きたか。

**入口②　うまくいきすぎているとき**
「なんで当たり前にできてるんだろう」と不思議に感じる場面。列に並ぶ、授業で手を挙げる、グループのLINEで誰かが先に話題を変える——なぜスムーズに進むのか。

**入口③　自分がやっていること**
自分が毎日やっている行為で「なんとなくそうしてる」もの。「なぜそうしているか」を説明できない行為こそ、EMのテーマになる。
:::

### テーマ候補の具体例

:::example 高校生が取り組みやすいテーマ例
| 場面 | 気になること（問いの種） |
|---|---|
| 購買・食堂の行列 | 列はどうやって「列」として維持されているか |
| 授業での挙手・指名 | 「指名されやすい座席」は存在するか、あるとしたらどのように作られているか |
| 部活の練習開始 | 練習開始の合図は誰がどのように出し、どのように受け取られているか |
| グループでの昼食 | 席のとり方・グループの境界はどう決まっているか |
| 廊下での擦れ違い | 知り合いと知らない人への視線や動作はどう異なるか |
| LINEグループでの既読スルー | 既読スルーの後に誰かが「返信しやすい状況」を作ることがあるか |
| コンビニの会計 | 店員と客のやりとりはなぜあのような決まった形をとるのか |
:::

## ステップ2　問いと方法を6要素で固める

:::concept ミニ実践計画の6要素
発表計画は以下の6点を1枚で示します。結論の良し悪しよりも、各要素が具体的かどうかが評価対象です。

| 要素 | チェックポイント |
|---|---|
| **①問い** | 「〜はどのように達成されているか」の形になっているか |
| **②方法** | 観察・転写・会話分析・MCAのどれを使うか明示されているか |
| **③データ範囲** | いつ・どこで・誰の・何分のデータかが具体的か |
| **④根拠** | どのデータ行からどう読んだかを行番号で示せるか |
| **⑤倫理** | 匿名化の方法・公開範囲・同意の有無が書かれているか |
| **⑥限界** | このデータだけでは分からないことを1点以上書けているか |
:::

## ステップ3　6要素の完成例を見る

### 完成例A：授業の挙手・指名場面

:::example テーマ「授業中の指名パターン」
**①問い**
教室の授業場面において、教師による「指名」はどのように組織されているか。特に、挙手なし指名が連続するときの順番パターンに着目する。

**②方法**
観察ログ（矢印記述）＋簡易転記。会話分析の「順序性」を分析枠組みとして使用。

**③データ範囲**
5限目・現代文の授業・教室前方から撮影した許可済みの観察メモ・約15分・参加者はA教師と生徒B〜G（匿名）。

**④根拠**
観察ログ3行目「AがBに指名→Bが答える→即座にCに指名」が3回連続した後、4回目でDが自発的に挙手した（7行目）。この変化が「連続指名のパターンが崩れた瞬間」として分析できる。

**⑤倫理**
・個人名・座席位置は記録しない（A教師・B〜G表記）
・メモの公開は授業内のみ、担当教員への提出に限定
・観察の事前承諾：授業担当教師に口頭で確認済み

**⑥限界**
このデータは1コマのみのため、このパターンが教師固有のものか、場面依存か、授業内容依存かは判断できない。複数コマの比較が必要。
:::

### 完成例B：購買の行列場面

:::example テーマ「購買の列の維持」
**①問い**
昼休みの購買において、「列」という秩序はどのような行為によって維持・修復されているか。特に、列への割り込みが起きたときの反応に着目する。

**②方法**
観察ログ（矢印記述）＋フィールドノート。背景期待とブリーチング分析の枠組みを使用。

**③データ範囲**
昼休み（12:30〜12:45）・購買前の廊下・目視観察メモ（録音なし）・参加者は観察できた並んでいる生徒A〜F（匿名）。

**④根拠**
観察ログ5行目「Cが列の途中に入る→後ろのDが体を前に寄せる（言語なし）→CがDに目をやる→Cが列の末尾に移動」。DとCの非言語的やりとりで修復が完了した。発話なしで秩序が回復したことが分析の焦点。

**⑤倫理**
・顔・制服・個人特定につながる情報は記録しない
・観察は公共空間（廊下）で行い、録音・録画なし
・分析結果の共有は授業内のみ

**⑥限界**
Cが末尾に移動した動機（Dの視線を見たから vs もとから並び直すつもりだった）はデータからは判断できない。
:::

## ステップ4　よくある詰まりと突破口

:::caution 詰まりポイントと対処法

**「データが取れない」**
→ 録音・録画は必須ではありません。観察メモと矢印記述だけでも分析できます。「5分間だけ、この場面を観察して書く」から始めてください。

**「分析できる気がしない」**
→ データ全体を分析しようとしないこと。「1つの隣接ペア」「1つの沈黙」「1つの修復」だけに絞れば、80〜120字の説明が書けます。

**「問いが決まらない」**
→ 先に「この場面の何が不思議か」を書き出す。「なぜ〜なのか」を「〜はどのように達成されているか」に変換するとEMの問いになります。

**「発表で何を言えばいいか分からない」**
→ 「問い→根拠（行番号）→結論→限界」の4点だけ言えばよい。スライドのデザインより先に、この4点が口で言えるか確認してください。

**「結論が出ない」**
→ 「分からなかった」も立派な結論です。「このデータでは〜が確認できなかった。それは〜という限界があるからで、次に〜を調べれば確認できるかもしれない」と書けば、分析として成立します。
:::

## ステップ5　サッチマンの視点：計画は台本ではない

:::concept サッチマン「計画と状況的行為」
ルーシー・サッチマンは「計画」を、行為の厳密な台本ではなく、状況の中で行為を調整するための**リソース（使い捨てるメモ）**として捉えました。

探究の発表計画も同様です。「完璧な計画を立ててから始める」のではなく、「計画を使いながら状況に応じて修正する」ことが実践の現実です。計画は完成品ではなく、「なぜこの方法でこのデータを見るのか」を説明できるドキュメントです。

→ 計画が変わっていいこと、限界があっていいことは「失敗」ではなく「EMの正直な記述」です。
:::

## 探究レポートの最小構成

発表・レポートに必要な最小の要素は以下の5段落です。

:::concept 探究レポート最小テンプレート
**段落1　問いと場面（3〜5文）**
「この探究では、〔場面〕において〔現象〕がどのように達成されているかを問いとした。対象は〔誰・いつ・どこ〕の〔何分〕のデータである。」

**段落2　方法（2〜3文）**
「〔観察メモ／転記〕を作成し、〔会話分析／ブリーチング分析〕の枠組みで分析した。転記記号は〔使用記号〕を用いた。」

**段落3　根拠と分析（5〜8文）**
「〔行番号〕において〔何が起きたか〕が確認できる。これは〔概念名〕の例として解釈できる。なぜなら〔根拠〕だからである。」

**段落4　結論（2〜3文）**
「以上のデータから、〔場面〕では〔結論〕と言える。」

**段落5　限界と展望（2〜3文）**
「ただし、このデータだけでは〔限界〕が確認できない。今後〔何〕を調べることで、より確かな記述が可能になると考える。」
:::

## やってみよう

:::exercise
1. 自分の「気になる場面」を1つ書く（テーマ発見の3つの入口を参考に）。
2. 6要素チェック表を埋める（空欄があれば、何が足りないかを書く）。
3. 「問い→根拠→結論→限界」の4点を口頭で30秒以内に言えるか試す。
4. 限界を1点書いて、「次に調べるとしたら何か」を1文追加する。
:::
