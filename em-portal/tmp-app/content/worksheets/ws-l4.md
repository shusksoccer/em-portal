## このワークの目的

会話が「ターンの連なり（順序性）」として組織されていることを、**隣接ペアの抽出**と**修復の追跡**という2つの作業で確かめる。

:::example この分析が見せてくれる驚き
誘いを断るとき、日本語でも英語でもアラビア語でも、人は同じ構造を使います——すぐに断らず、まず**間を置き**、**前置きをして**から断る（「うーん…ちょっと用があって」）。この「普遍的な断り方の文法」は、誰にも教わっていないのに世界共通で観察されます。さらに驚くべきことに、修復（repair）の分析は「会話は互いに協力して成立している」ことを示します。相手の言い間違いをわざわざ指摘して直させる「他者修復」より、自分で気づいて直す「自己修復」のほうが圧倒的に多い——これは人が無意識に相手のメンツを守っているからです。
:::

## Part 1　隣接ペア・ハント

### 隣接ペアとは

:::concept
**隣接ペア（Adjacency Pair）**は、2つのターンで1セットを作る会話の基本単位です。

| 名前 | 記号 | 例 |
|---|---|---|
| 第一ペア部分（FPP） | → | 「質問」「依頼」「挨拶」「申し出」 |
| 第二ペア部分（SPP） | → | 「答え」「応答（承諾/断り）」「挨拶返し」「受け入れ/辞退」 |

**大事なポイント：** FPPが出ると、SPPが「次に来るべきもの」として条件づけられます。もしSPPが来ない（沈黙・話題転換・確認質問）場合、それ自体が「問題」として処理されます。
:::

:::example 選好組織（Preference Organization）
第二ペア部分には「好まれる応答（選好対）」と「好まれない応答（非選好対）」があります。

| FPP | 選好対（Preferred）| 非選好対（Dispreferred）|
|---|---|---|
| 誘い | 「うん、行く」（即答・短い） | 「うーん…ちょっと用があって」（遅延・言い訳・謝罪） |
| 依頼 | 「いいよ」 | 「難しいんだけど…」 |
| 評価 | 「そうだよね」（同意） | 「えーでも…」（不同意） |

非選好対は遅延・ためらい・前置きを伴うことが多く、それ自体がデータになります。
:::

### 練習用データ（教師配布版）

```
1:  A「今日の昼、一緒に食べない?」
2:  (1.5)
3:  B「あー、今日ちょっと用があって」
4:  A「そっか」
5:  B「明日は大丈夫だよ」
6:  A「ほんとに? じゃあ明日」
7:  B「うん、12時に食堂で」
8:  A「わかった、また連絡して」
9:  B「りょー」
```

### 手順（生徒向けステップバイステップ）

**ステップ1**　転記を読み、場面を1行で要約する（例：昼食の約束）。

**ステップ2**　「ペアになりそうな箇所」に○を付ける（誘い→応答、質問→答えなど）。

**ステップ3**　○の中で、第一ペア部分（FPP）と第二ペア部分（SPP）にラベルを書く。

**ステップ4**　第二ペア部分が遅れる／変形する箇所に ★ を付ける（沈黙・確認・言い換えなど）。

**ステップ5**　★の後に何が起きて会話が進んだかを、行番号つきでメモする。

**ステップ6**　連鎖図を描く：FPP → SPP の矢印を3組（行番号を入れる）。

**ステップ7**　面白い連鎖を1つ選び、「この連鎖は何を達成しているか」を80〜120字で書く（行番号を1つ以上含める）。

**ステップ8**　班で「別の切り方（別解）」を1つ出す（断定しない練習）。

### 記入欄

**場面要約（1行）：**

**抽出した隣接ペア（行番号）：**
- ① FPP ＿行 → SPP ＿行（ペアの種類：　　）
- ② FPP ＿行 → SPP ＿行（ペアの種類：　　）
- ③ FPP ＿行 → SPP ＿行（ペアの種類：　　）

**★ 遅れ・変形があった箇所（行番号）：**
- 行番号：　　　／何が起きた：

**連鎖図（矢印で）：**


## Part 2　修復トレース

### 修復（Repair）とは

:::concept
**修復（repair）**は、発話の「産出・理解・聞き取り」に関わるトラブルに対処する方法です。失敗や謝罪ではなく、会話を続けるための仕組みです。

| 段階 | 意味 |
|---|---|
| トラブル源 | 問題が起きた箇所（言い間違い・聞き取れなかった部分など） |
| 修復開始 | トラブルが「問題」として示された箇所（「え？」「え、なんて？」など） |
| 修復実行 | 問題が解決された箇所（言い直し・言い換え・説明など） |
| 復帰 | 会話が元のトピックへ戻った箇所 |

**自己 vs 他者：** 修復を「開始」するのが話している本人（自己）か相手（他者）か、「実行」するのが本人か相手か、4通りの組み合わせがあります。
:::

### 練習用データ（教師配布版）

```
1:  A「土曜に図書館、あ、図書室でやるって言ってた?」
2:  B「え、なんて?」
3:  A「土曜日に集まる場所って図書室だっけ?」
4:  B「あー、図書室じゃなくて多目的室だよ」
5:  A「多目的室か、わかった」
6:  B「3時からね」
7:  A「3時ね、ありがとう」
```

### 手順（生徒向けステップバイステップ）

**ステップ1**　転記を読み、「言い直し・聞き返し・確認・言い換え」に下線を引く。

**ステップ2**　下線の直前を「トラブル源」、下線部分を「修復開始」、解決した部分を「修復実行」として区切る。

**ステップ3**　修復表テンプレ（下記）に、行番号で埋める。

**ステップ4**　「誰が修復を開始したか」「誰が実行したか」を書く（自己 or 他者）。

**ステップ5**　1か所選び、「なぜ会話が続いたと言えるか」を根拠（行番号）つきで2文で説明する。

**ステップ6**　班で比較し、修復の型（自己開始自己実行・他者開始自己実行など）の違いを共有する。

### 修復表

| 項目 | 行番号 | 内容 |
|---|---|---|
| トラブル源 | | |
| 修復開始 | | |
| 修復実行 | | |
| 復帰 | | |
| 開始したのは（自己 / 他者） | | |
| 実行したのは（自己 / 他者） | | |

**根拠つき2文説明（行番号必須）：**

**班での修復タイプ比較：**
- 自己開始自己実行の例 → ＿行目「　　　　　」
- 他者開始自己実行の例 → ＿行目「　　　　　」

## 解答例（練習用サンプルデータを使用）

> ※授業で配布されたサンプルデータに対する解答例です。

### Part 1　隣接ペア解答例

```
1:  A「今日の昼、一緒に食べない?」
2:  (1.5)
3:  B「あー、今日ちょっと用があって」
4:  A「そっか」
5:  B「明日は大丈夫だよ」
6:  A「ほんとに? じゃあ明日」
7:  B「うん、12時に食堂で」
8:  A「わかった、また連絡して」
9:  B「りょー」
```

**場面要約：** 昼食の誘いと代替日の調整

**抽出した隣接ペア（行番号）：**
- ① FPP 1行 → SPP 3行（誘い → 断り）※1.5秒の遅延あり＝非選好対の型
- ② FPP 5行 → SPP 6行（代替提案 → 受け入れ）
- ③ FPP 8行 → SPP 9行（依頼「連絡して」 → 承諾「りょー」）

**★ 遅れ・変形があった箇所：**
- 行番号：2行目 / 何が起きた：1.5秒の沈黙。誘いへの即答を避け、断りにくさを示している

**連鎖図：**
```
1行：FPP（誘い）      →  3行：SPP（断り・非選好対）
5行：FPP（代替提案）  →  6行：SPP（受け入れ）
8行：FPP（依頼）      →  9行：SPP（承諾）
```

**説明（80〜120字）：**
1行目の誘い（FPP）に対し、3行目のBの応答は1.5秒（2行目）の沈黙を挟んでいる。即答せず理由を述べる形は非選好対の典型で、断りにくい状況を示している。5〜9行目では代替案によって誘いが形を変えて達成されている。

**班での別解：**
1.5秒の沈黙はBが本当に用を思い出していただけで、断ることを迷っていたわけではない可能性がある。

### Part 2　修復トレース解答例

```
1:  A「土曜に図書館、あ、図書室でやるって言ってた?」
2:  B「え、なんて?」
3:  A「土曜日に集まる場所って図書室だっけ?」
4:  B「あー、図書室じゃなくて多目的室だよ」
5:  A「多目的室か、わかった」
```

**修復表：**

| 項目 | 行番号 | 内容 |
|---|---|---|
| トラブル源 | 1行目 | 「図書館」という言い間違い |
| 修復開始 | 1行目 | 「あ、」という自己中断と言い直し |
| 修復実行 | 1行目 | 「図書室」への訂正（自己修復） |
| 復帰 | 1行目後半 | 「図書室でやるって言ってた？」と本来の質問に戻る |
| 開始したのは | 自己（A） | |
| 実行したのは | 自己（A） | |

**根拠つき2文説明：**
1行目でAは「図書館」と言いかけた直後に「あ、」と中断し、自ら「図書室」に修正した（自己開始自己実行）。この自己修復があったため会話は中断されず、Bの質問（2行目）はAの言い間違いではなく別の理由（聞き取れなかった）で起きたことが分かる。

**班での修復タイプ比較：**
- 自己開始自己実行の例 → 1行目「図書館、あ、図書室で」
- 他者開始自己実行の例 → 2行目「え、なんて？」→ 3行目でAが言い換え

## 評価基準（ルーブリック）

| 観点 | 3（できる） | 2（だいたい） | 1（これから） |
|---|---|---|---|
| 抽出の妥当性 | FPP/SPPが筋の通る形で複数示せる | 一部は示せる | ペアの基準が不明 |
| 順序性の理解 | 遅れ・挿入などを行番号で説明できる | 基本形は説明できる | 順序として捉えられない |
| 修復の同定 | トラブル源→開始→実行→復帰を行番号で追える | 一部曖昧だが追える | 区切りが不適切 |
| 概念の区別 | 修復の開始と実行、自己と他者を区別できる | 部分的に区別 | 区別できない |
| 根拠提示 | 行番号を示して説明できる | 説明はあるが根拠が弱い | 根拠がなく感想中心 |

---

## 注意

修復は「謝罪」や「仲直り」ではなく、あくまで会話の進行を保つ仕組みです。心理的な評価（「Bは嫌がっていた」など）ではなく、行番号で示せる事実（「2行目でBが聞き返したことで、4行目でAが言い換えた」）として説明してください。
