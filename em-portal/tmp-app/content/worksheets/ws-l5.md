## このワークの目的

模擬データを使って、**背景期待（Background Expectancies）がどこで崩れ、どう反応され、どう回復するかを分析する**。実際に違背実験を行うのではなく、データを読んで分析することが活動の核心です。

:::example この分析が見せてくれる驚き
ガーフィンケルの実験で最も注目すべきは、被験者が**怒った後すぐに「説明」を求めた**ことです。「なんで？」「どうしたの？」——人はルールが崩れると、ただ困惑するのではなく、**秩序を取り戻そうとする力を自動的に発動させます**。この「修復への衝動」こそが社会の秩序を支えているのです。また、実験が終わり「冗談だった」と告げられると、被験者の怒りはすぐに笑いに変わります。そのスピードも、社会秩序がいかに瞬時に再構築されるかを示しています。
:::

## 背景期待とブリーチングとは

:::concept
**背景期待（Background Expectancies）**とは、その場で「当然のこと」として共有されている前提のことです。ルールとして書かれていないのに、それが崩れると参加者が困惑・不快・怒りを示します。

**ガーフィンケルの違背実験（Breaching Experiment）**は、この見えない前提を「意図的に崩す」ことで可視化する手法です。崩れたときの相手の反応が、前提が何であったかを教えてくれます。

分析の順序：
1. **期待の特定**：その場で「当然」とされていた前提は何か
2. **逸脱の記録**：どの行動・発話がその前提を崩したか（行番号で）
3. **反応の記録**：相手はどう反応したか（困惑・怒り・修復要求など）
4. **回復の記録**：どのように元の秩序に戻ったか
:::

---

## ガーフィンケルの4つの実験類型

| 類型 | 崩された前提 | 相手の典型的反応 |
|---|---|---|
| 意味の執拗な問い直し | 日常言語は文脈で意味が共有される | 「なんか怒ってる？」「どうしたの？」と困惑 |
| 家庭内での「ゲスト化」 | 家族は非公式・親密に接する | 「何かあった？」と修復を試みる |
| ゲームルールの破壊 | ゲームの共有は共同作業で成立する | 抗議・怒り・対話が止まる |
| 役割カテゴリーの誤認 | 外見や場所から役割を瞬時に割り当てる | 「私に言ってる？」「店員じゃないです」 |

## 分析用模擬データA（挨拶の崩れ）

```
1:  A「おはよう」
2:  B「"おはよう"とはどういう意味ですか？具体的に定義してください」
3:  A「え…(2.0)なんか機嫌悪い？」
4:  B「質問に答えてください。"おはよう"の意味を教えてください」
5:  A「なんか変なことを言ってる…(.)友達に聞いてみる」
6:  B「ごめん、冗談だよ。おはよう」
7:  A「え、何だったの？（笑）」
```

> 1行目の「おはよう」にはどんな背景期待があるか。2行目の発話で何が崩れたか。3〜5行目の反応はどの期待が崩れたことを示しているか、行番号で書いてみよう。

## 分析用模擬データB（役割カテゴリーの誤認）

```
1:  （Aはエプロンなし、店内に買い物に来た一般客）
2:  B「すみません、この棚の在庫ってありますか？」
3:  A「え？(1.2)私、お客さんですよ」
4:  B「あ、失礼しました。店員さんかと思って」
5:  A「大丈夫です。でも店員さん呼びましょうか？」
6:  B「あ、はい、すみません」
```

> どのカテゴリー（役割）の前提が崩れたか。3行目でAが示した反応は何を意味しているか。4〜6行目で秩序がどう回復したかを行番号で書いてみよう。

## 安全ルール確認

:::caution
**重要：このワークは「分析のみ」です。**

- 実際に他者を困惑させる実験を行わない
- 「面白いから実際にやってみる」は禁止
- 模擬データ・架空例のみを使って分析する
- 発表時に「実施を推奨しない」ことを必ず明記する
:::

安全チェック（提出前に確認）：

## 解答例（模擬データAを使用）

> ※「おはよう」データに対する解答例です。

```
1:  A「おはよう」
2:  B「"おはよう"とはどういう意味ですか？具体的に定義してください」
3:  A「え…(2.0)なんか機嫌悪い？」
4:  B「質問に答えてください。"おはよう"の意味を教えてください」
5:  A「なんか変なことを言ってる…(.)友達に聞いてみる」
6:  B「ごめん、冗談だよ。おはよう」
7:  A「え、何だったの？（笑）」
```

**特定した背景期待（3つ）：**
- 期待①：挨拶には挨拶が返ってくる（挨拶の返礼）
  - 根拠行：3行目でAが「機嫌悪い？」と困惑した → 返礼が来なかったため
- 期待②：挨拶の意味は文脈で自明で、定義を求めるものではない
  - 根拠行：5行目でAが「変なことを言ってる」と評価した → 意味への問い直し自体が異常と感じられた
- 期待③：やり取りは短く完了し、次の話題へ進む
  - 根拠行：4行目でBが再び定義を要求し、やり取りが続いた → 通常の終了が妨げられた

**逸脱→反応→回復の流れ：**

| 段階 | 行番号 | 内容 |
|---|---|---|
| 逸脱が起きた行 | 2行目 | 挨拶への返礼の代わりに意味の問い直しが発話された |
| 相手の反応 | 3行目 | 「え…」という沈黙と「機嫌悪い？」という修復要求 |
| 修復要求または回避 | 5行目 | 「友達に聞いてみる」という場からの離脱 |
| 回復が見える行 | 6〜7行目 | Bが「冗談だよ」と説明し、「おはよう」と言い直した後にAが笑い → 通常の挨拶関係に復帰 |

## 評価基準（ルーブリック）

| 観点 | 3（できる） | 2（だいたい） | 1（これから） |
|---|---|---|---|
| 期待の特定 | 背景期待を3つ具体的に書けた | 2つは書けた | 一般論で終わった |
| 根拠提示 | 行番号つきで逸脱・反応・回復を説明できた | 一部は行番号あり | 行番号なし・感想中心 |
| 安全配慮 | 分析と実施の区別を明確に理解している | 理解しているが確認不足 | 区別できていない |
| 説明の明確さ | 期待→逸脱→反応→回復の流れを説明できた | 大部分は説明できた | 流れが追えていない |

---

## 注意

「Bは意地悪だった」「Aは不安になった」のような心理的断定ではなく、「2行目でBが意味を問い返したことで、3行目のAが困惑した」のように、**行番号を根拠にした事実の記述**を心がけてください。
